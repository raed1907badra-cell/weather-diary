# Weather Diary — Дневник погоды

GUI-приложение для ведения дневника погодных наблюдений с сохранением данных в JSON.

## Автор
Бугай Александр Сергеевич

## Описание программы
Приложение "Weather Diary" позволяет пользователю вести учёт погодных условий. Реализован удобный графический интерфейс на библиотеке Tkinter. Данные сохраняются в JSON-файл и загружаются при следующем запуске.

### Основные функции:
- Добавление записей о погоде (дата, температура, описание, наличие осадков)
- Просмотр всех записей в таблице
- Фильтрация записей по дате и минимальной температуре
- Удаление выбранных записей
- Автоматическое сохранение данных в JSON
- Валидация вводимых данных с выводом понятных сообщений об ошибках

## Требования
- Python 3.6 или выше
- Библиотека Tkinter (входит в стандартную поставку Python)
- Дополнительные библиотеки не требуются

## Как запустить приложение

### Способ 1: Клонирование репозитория
```bash
git clone https://github.com/raed1907badra-cell/weather-diary.git
cd weather-diary
python main.py
Способ 2: Скачивание ZIP-архива
Скачайте архив с GitHub (кнопка "Code" → "Download ZIP")

Распакуйте архив в любую папку

Откройте терминал в этой папке

Выполните команду:

bash
python main.py
Примеры использования
Добавление записи о погоде
Дата: 15.03.2024

Температура: -2.5

Описание: Пасмурно, небольшой снег

Осадки: отметить галочку

Нажать кнопку "Добавить запись"

Результат: запись появится в таблице и сохранится в JSON-файл.

Фильтрация записей
Пример 1 — фильтр по дате:

В поле "Дата" ввести: 15.03.2024

Нажать "Применить фильтр"

Отобразятся только записи за указанную дату

Пример 2 — фильтр по температуре:

В поле "Температура выше" ввести: 10

Нажать "Применить фильтр"

Отобразятся записи с температурой ≥ 10°C

Пример 3 — комбинированный фильтр:

Дата: 20.06.2024

Температура выше: 25

Будут показаны записи за 20.06.2024 с температурой от 25°C

Удаление записи
Кликнуть по нужной строке в таблице

Нажать "Удалить выбранную запись"

Подтвердить удаление

Валидация ввода
Приложение проверяет корректность данных и выводит сообщения об ошибках:

Дата должна быть в формате ДД.ММ.ГГГГ

Температура должна быть числом (допустимы отрицательные значения и десятичные дроби)

Описание погоды не может быть пустым

Формат хранения данных (JSON)
json
[
  {
    "date": "15.03.2024",
    "temperature": -2.5,
    "description": "Пасмурно, небольшой снег",
    "precipitation": "Да"
  },
  {
    "date": "20.06.2024",
    "temperature": 28.0,
    "description": "Солнечно, ясно",
    "precipitation": "Нет"
  }
]
Тестирование
Позитивные тесты (ожидаемый результат — успех)
Добавление записи: 01.01.2024, температура -10, описание Морозно

Добавление записи с дробной температурой: 23.5

Фильтр по дате, на которую есть записи

Фильтр по температуре 0 — показывает все записи с температурой ≥ 0

Негативные тесты (ожидаемый результат — ошибка)
Дата 32.13.2024 → сообщение о неверном формате

Температура abc → сообщение "должна быть числом"

Пустое описание → сообщение "не может быть пустым"

Граничные тесты
Температура 0 → принимается

Температура -273 → принимается

Високосная дата 29.02.2024 → принимается

Дата 29.02.2023 → ошибка (2023 не високосный)

Описание длиной 200 символов → принимается
