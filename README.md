# Weather Diary — Дневник погоды

GUI-приложение для ведения дневника погодных наблюдений с сохранением данных в JSON.

## Автор
Бугай Александр Сергеевич

## Описание программы
Приложение "Weather Diary" позволяет пользователю вести учёт погодных условий. Реализован удобный графический интерфейс на библиотеке Tkinter. Данные сохраняются в JSON-файл и загружаются при следующем запуске.

Основные функции:
- Добавление записей о погоде (дата, температура, описание, наличие осадков)
- Просмотр всех записей в таблице
- Фильтрация записей по дате и минимальной температуре
- Удаление выбранных записей
- Автоматическое сохранение данных в JSON
- Валидация вводимых данных с выводом понятных сообщений об ошибках

## Требования
- Python 3.6 или выше
- Библиотека Tkinter (входит в стандартную поставку Python)
- Дополнительные библиотеки не требуются

## Как запустить приложение

Способ 1: Клонирование репозитория

git clone https://github.com/raed1907badra-cell/weather-diary.git

cd weather-diary

python main.py

Способ 2: Скачивание ZIP-архива

Скачайте архив с GitHub (кнопка Code, затем Download ZIP). Распакуйте архив в любую папку. Откройте терминал в этой папке. Выполните команду python main.py

## Примеры использования

Добавление записи о погоде:
Введите дату 15.03.2024, температуру -2.5, описание "Пасмурно, небольшой снег", отметьте галочку "Осадки", нажмите кнопку "Добавить запись". Результат: запись появится в таблице и сохранится в JSON-файл.

Фильтрация записей:
Фильтр по дате — в поле "Дата" введите 15.03.2024, нажмите "Применить фильтр", отобразятся только записи за указанную дату. Фильтр по температуре — в поле "Температура выше" введите 10, нажмите "Применить фильтр", отобразятся записи с температурой от 10 градусов. Комбинированный фильтр — укажите дату 20.06.2024 и температуру выше 25, будут показаны записи за 20.06.2024 с температурой от 25 градусов.

Удаление записи:
Кликните по нужной строке в таблице, нажмите "Удалить выбранную запись", подтвердите удаление.

## Валидация ввода
Приложение проверяет корректность данных и выводит сообщения об ошибках. Дата должна быть в формате ДД.ММ.ГГГГ. Температура должна быть числом, допустимы отрицательные значения и десятичные дроби. Описание погоды не может быть пустым.

## Формат хранения данных (JSON)
Файл weather_data.json создаётся автоматически при первом запуске приложения. Пример структуры данных:

{
  "date": "15.03.2024",
  "temperature": -2.5,
  "description": "Пасмурно, небольшой снег",
  "precipitation": "Да"
}

## Тестирование

Позитивные тесты (ожидаемый результат — успех):
Добавление записи 01.01.2024 с температурой -10 и описанием "Морозно". Добавление записи с дробной температурой 23.5. Фильтр по дате на которую есть записи. Фильтр по температуре 0 показывает все записи с температурой от 0.

Негативные тесты (ожидаемый результат — ошибка):
Дата 32.13.2024 вызывает сообщение о неверном формате. Температура abc вызывает сообщение "должна быть числом". Пустое описание вызывает сообщение "не может быть пустым".

Граничные тесты:
Температура 0 принимается. Температура -273 принимается. Високосная дата 29.02.2024 принимается. Дата 29.02.2023 вызывает ошибку так как 2023 не високосный. Описание длиной 200 символов принимается.
